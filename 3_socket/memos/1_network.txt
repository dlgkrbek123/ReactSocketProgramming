---------------------------------------------------------------------------------

OSI 7 Layer
=> 네트워크 통신사들이 방식이 서로달라 문제발생
=> 그래서 국제표준화기구가 결정한 프로토콜과 모델
=> 송신할 때 7개의 단계를 거치면서 헤더정보가 추가됨

---------------------------------------------------------------------------------

TCP/IP
=> 어떻게 원하는 호스트에게 안정적으로 데이터 전송을 할까
=> OSI 7 Layer의 실무적이고 단순화한 모델 / 프로토콜

TCP : 신뢰성있는 데이터 전송
IP  : 기기간의 빠른 데이터 전송

---------------------------------------------------------------------------------

소켓 통신
=> TCP나 UDP를 사용하는 두 기기간의 "연결" (양방향)
=> IP 주소와 포트번호를 이용해 연결을 유지
=> 실시간 통신을 위해 연결이 필수

3way-handshake를 통해 연결을 수립

---------------------------------------------------------------------------------

HTTP를 통한 양방향 통신

폴링 : 클라이언트에서 주기적으로 서버에 request (과부하)
롱 폴링 : 서버에서 이벤트가 발생하면 클라이언트에 응답을 주는 방식
스트리밍 : 지속적인 연결상태로 서버의 데이터를 클라이언트에 전달
