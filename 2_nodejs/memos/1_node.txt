js는 브라우져를 초월해서 사용된다.
=> 크롬이 v8엔진 탑재
=> 이를 기반으로 자바스크립트 런타임인 nodejs를 구현
=> 자바스크립트를 실행하는 환경

---------------------------------------------------------------------------------

=> 기존 웹서버들은 요청을 동기방식으로 처리
=> 순차적이라 이전의 일이 끝나야 다음 일을 처리
=> 병목이발생 (IO 블로킹)
=> 한 줄의 햄버거 주문 대기열
=> 다음사람은 이전사람의 주문이 끝나기를 기다려야함

=> 이를 해결하기 위해 멀티스레드를 사용하여 병렬 처리 (주문열 증설)
=> 스레드는 자원을 소모, 공통 메모리는 어떻게 관리?

---------------------------------------------------------------------------------

nodejs는 네트워크/IO처리에 특화
=> 단일 스레드의 이벤트 기반 비동기 방식으로 동작
=> 이벤트 기반 : 이벤트와 콜백을 통해 작업을 처리
=> 이벤트가 발생하면 콜백함수가 호출되어 처리
=> 여러명이 동시에 주문을 하고 각 주문은 개별적으로 처리될 수 있음 (휴대폰 오더)
